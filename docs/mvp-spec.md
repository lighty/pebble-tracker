# Pebble Tracker MVP仕様

## 目的

Pebble Tracker は、ユーザーが任意のイベントを素早く記録し、あとから日単位で振り返れる Obsidian プラグインである。

MVP では「回数を記録するイベント」に限定し、最初の代表ユースケースとしてトイレ記録を想定する。ただしプラグイン自体はトイレ専用ではなく、ユーザーが GUI からイベント定義を追加できる構成にする。

## 基本方針

- スマートフォン利用を優先する
- 最重要機能は「素早く記録すること」
- 記録画面を主画面にする
- 集計画面は別画面として用意する
- イベント定義は GUI から追加・編集・削除できる
- 記録はワンタップで即保存する

## 導線

- コマンドパレットに `Open Pebble Tracker` を追加する
- 左リボンに Pebble Tracker のアイコンを追加する
- どちらの導線でも専用タブの記録画面を開く
- 前回選択していたイベントを復元する

## 画面構成

MVP では以下の 3 つを提供する。

1. 記録画面
2. 集計画面
3. イベント追加・編集 UI

## 記録画面

記録画面は日常利用の主画面であり、スマートフォンでの全画面表示を前提とする。

### 表示要素

- 現在選択中のイベント名
- `イベント切替` アクション
- `集計` アクション
- メモ入力欄
- 大きな `記録する` ボタン
- 直近の記録一覧

### 挙動

- 記録対象は常に 1 イベントだけ選択されている
- `記録する` を押すと、選択中イベントに対して現在時刻で即保存する
- メモ入力がある場合は一緒に保存する
- 保存後はメモ欄をクリアする
- 保存完了後に軽い通知を表示する
- 直近の記録一覧には最新数件を表示する

### メモ欄

- メモ欄は記録ボタンの近くに常時表示する
- イベント定義の `allowMemo` が `false` の場合は非表示または無効化する

## イベント切替 UI

イベント一覧は常時表示しない。記録画面から明示的に開く UI とする。

### 想定形態

- スマートフォンではボトムシート風 UI を優先する

### 表示要素

- イベント名
- アイコン
- 色
- `+ イベント追加`
- 各イベントの `編集`
- 各イベントの `削除`

## 集計画面

集計画面は、選択中イベント 1 件のみを対象に表示する。

### 表示要素

- 対象イベント名
- 期間切替: `7日 / 30日 / 90日 / 全期間`
- 日別・週別・月別の棒グラフ
- 日別・週別・月別の一覧
- 選択中期間の記録一覧

### 集計一覧

- 各日付ごとの記録回数を表示する

### 記録一覧

- 時刻
- メモ

## イベント定義

MVP のイベント定義は以下の項目を持つ。

- `id`
- `name`
- `icon`
- `color`
- `allowMemo`

### 補足

- 並び順は MVP では作成順とする
- 将来的な入力タイプ拡張は考慮するが、MVP では UI に出さない

## 記録データ

1 件の記録は以下の項目を持つ。

- `id`
- `eventTypeId`
- `timestamp`
- `memo?`

## 削除仕様

- イベント削除時は確認ダイアログを表示する
- ダイアログには「関連する記録も一緒に削除される」ことを明記する
- 確認後は関連記録も含めて完全削除する

## 保存方式

- イベント定義と選択状態は Vault 内の JSON ファイルとして保存する
- 既定パスは `PebbleTracker/settings.json` とする
- 記録レコードは Vault 内の CSV ファイルとして保存する
- 既定パスは `PebbleTracker/records.csv.md` とする
- CSV はユーザーが後から再利用・編集しやすいことを優先する

## MVP で扱わないもの

- 数値入力イベント
- 選択肢入力イベント
- 複数イベントの比較表示
- Dataview 連携
- エクスポート / インポート
- カレンダーヒートマップ
- 並び替え UI
